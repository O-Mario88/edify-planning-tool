# Edify Planning Tool — UI & Functionality Audit (Pre-Pilot)

**Date:** 2026-07-08
**Scope requested:** dead space, dead buttons, invisible buttons, content overlap in cards/tiles, filter functionality, feature functionality — ahead of a first test rollout to a small user group.

**How it was audited:** the app was booted locally (Postgres 16, `seed --demo`: 700 schools, 260 activities, 32 users), then crawled with a headless browser as **all 10 demo roles across 172 page visits** (with in-page detectors for overlap, clipped text, invisible interactive elements, dead space, console errors, and failed requests, at desktop/tablet/phone widths). In parallel, all **235 templates**, the CSS bundle, and every frontend view were statically analyzed for unwired controls, broken routes, and filter → queryset wiring.

---

## Executive summary

**The app is not ready for a user pilot in its current state — but it is close, and most symptoms trace back to a small number of systemic root causes.** The page architecture (drawer system, HTMX wiring, pagination/filter preservation on the major directories) is fundamentally sound. What breaks the experience is: a stale compiled CSS bundle that renders ~25 primary buttons invisible, four server crashes on core pages (one of which bricks the Country Director role entirely), two flagship dashboards whose numbers are hardcoded mockups, and a long tail of dead buttons and no-op filters.

**Recommendation: run a scoped pilot (CCEO + Program Lead journeys) after a P0 fix sprint of roughly 3–5 engineering days.** Hold the Accountant, IA-dashboard, and CD-dashboard surfaces back until their mock data and crashes are resolved. Details in §9.

### The five root causes

| # | Root cause | Symptoms it explains |
|---|---|---|
| 1 | **Stale Tailwind build, no CSS pipeline in repo.** `static/css/main.css` is a committed Tailwind v4.3.2 artifact compiled from an older template set; there is no `package.json`/`tailwind.config` anywhere to regenerate it. **472 of 1,222 distinct utility classes used in templates are missing from the bundle** (e.g. `bg-slate-800`, `text-[#2e505f]` ×74, `w-80`/`h-80`, `max-w-[1600px]` ×14 pages, `hover:bg-blue-700` ×44). | Invisible white-on-white buttons everywhere, missing hover states, unconstrained page widths (dead space), collapsed chart containers. |
| 2 | **Never-exercised code paths that crash.** Four page views query model fields that don't exist. | `/dashboard` 500 for CountryDirector; `/staff` and `/leave-requests` 500 for Admin & HR; `/coverage` 500 for Program Lead. |
| 3 | **Mockup pages shipped as if real.** The IA Quality Dashboard and Accountant Fund Disbursement Dashboard render hardcoded numbers and fake records, with entire action surfaces (31 controls on IA dashboard alone) not wired to anything. | Fake KPIs ("UGX 2.48B", "2,917 verified"), fake fund requests mixed into the real queue, alert()-only "Save" buttons. |
| 4 | **Sidebar/permission drift.** The sidebar's role conditionals don't match the backend `can_view_page` matrix, so roles see nav items that bounce them back with "Access Denied". | IA: 11 of 24 sidebar links bounce; Partner: 10 of 21; Accountant: 9; RVP: 4; HR: 3. |
| 5 | **All interactivity depends on 4 external CDNs** (HTMX, Alpine, ApexCharts, Flowbite from unpkg/jsdelivr/cloudflare + Google Fonts) with no local fallback. When the CDN is unreachable — a real scenario for field users on poor connectivity — pages render but **every drawer, dropdown, chart, tab, and filter dies silently.** (Empirically confirmed: this environment blocks those CDNs and the app lost all interactivity until the libraries were served locally.) | Total interactivity failure on bad networks; slow first paint. |

## 1. Feature functionality — server crashes (P0)

All four are `FieldError`/`AttributeError` querying fields that have never existed — these paths were never run end-to-end.

| Page | Roles affected | Error | Location |
|---|---|---|---|
| `/dashboard` | **CountryDirector (hard-bricked: error page has no nav, so the role can't go anywhere)** | `FieldError: Cannot resolve keyword 'staff_assignments'` — School has no such relation; `StaffSchoolAssignment.school_id` is a plain CharField (no FK/reverse). Crashes whenever any Program Lead has supervisees. | `apps/frontend/views/dashboard_views.py:152` |
| `/staff` | Admin, HR | `AttributeError: 'StaffProfile' object has no attribute 'location_name'` | `apps/frontend/views/staff_views.py:90` |
| `/leave-requests` | Admin, HR | `FieldError: Cannot resolve keyword 'user'` (field is `staff`) | `apps/frontend/views/extended_views.py:596` |
| `/coverage` | Program Lead | `FieldError: Cannot resolve keyword 'schools'` on Cluster | coverage view (`apps/frontend/views/`) |

Also in this class (latent): the IA queue view filters `Activity.objects.filter(month=…)` but `Activity` has no `month` field (`ia_views.py:69-70`) — any request supplying `month` will 500.

**Broken workflow — Champion review is 100% unreachable:** the catch-all route `core-schools/<str:plan_id>` is registered before `core-schools/champions` and `core-schools/champion-candidates` (`apps/frontend/urls.py:292-297`), so those URLs resolve to the wrong view; and the champion templates use six un-namespaced `{% url %}` tags (`'core_schools'`, `'champion_candidates'`, `'champions_list'`, `'champion_review_drawer'`, `'champion_reject_action'`, `'champion_approve_action'`) that raise `NoReverseMatch` because `apps/frontend/urls.py:11` sets `app_name = "frontend"` — rendering would 500 anyway (`templates/pages/core_schools/champions.html:13,23`, `champion_candidates.html:13,23,78`, `partials/core_schools/champion_review_drawer.html:177,187`).

**Broken page — Accountant dashboard interactive layer is dead:** `templates/pages/accounts/dashboard.html:9` injects `{{ all_funds_json|safe }}` (JSON full of `"` quotes) inside a **double-quoted** `x-data="…"` attribute. The attribute terminates at the first quote; the browser logs `SyntaxError` + `ReferenceError: allFunds/selectedItem/activeTab/formatCurrency is not defined`, and the fund list, detail pane, tabs, and currency formatting never initialize. Fix: `{{ all_funds|json_script:"funds-data" }}` + `x-data` reading from the script tag, or single-quote the attribute and escape properly.

**Mock data baked into real screens (P0 for a finance/M&E tool):**
- Accountant dashboard: 5 fabricated fund requests ("Sarah M.", "Ruth W."…, FR-2481…) are **concatenated with the real queue** (`finance_operating_views.py:98-226`, comment: "mockup acts as backup/additional mockups for full visuals") and all 8 KPIs are hardcoded strings ("UGX 2.48B", "78%") (`:230-238`).
- IA Quality Dashboard (`/ia/dashboard/`): entire page is a static mockup — hardcoded work queue, alerts, charts, date range "May 19 – May 25, 2025" (`ia_views.py:365-499`), with **31 dead controls** including per-row "Verify"/"Return", queue tabs, pagination, and six upload CTAs (`templates/pages/ia/analytics_dashboard.html:41-705`).
- Also: CCEO fund queue mock (`budget_views.py:496`), agenda fallback mock (`dashboard_views.py:483`), default mock permissions on the roles page (`extended_views.py:734`).
- On-screen contradiction on the main dashboard: "Upcoming Today: 2" next to "Today's Priorities: 0 — No priorities scheduled" (two different data sources).

**Fake-success buttons — worst category for user trust (P0):**
- Admin → Roles & Permissions → **"Save System Changes"** shows `alert('Role permissions saved successfully!')` and saves nothing (`templates/pages/admin/roles_permissions.html:24`); "Restore Defaults" same (`:18`).
- "Run Quality Scan" (`admin/data_quality_center.html:15-17`), Analytics "Export" claims a CSV is downloading (`analytics/index.html:16-17`), "Export Audited PDF Snapshot" claims an email is being sent (`partials/analytics/donor_reporting_snapshot.html:44-46`), "View Action Plan" claims slots were exported (`partials/analytics/recommended_insights.html:68-70`).

**Global search is dead on every page except `/schools`:** the topbar input ("Search schools, staff, districts…") only gets its HTMX wiring when `request.path == '/schools'` (`templates/layouts/shell.html:94-103`). Everywhere else, typing + Enter does nothing. The dedicated `/search` page works but **nothing links to it** (orphaned).

**Exports:** Schools "Export to Excel/CSV" — `export` param is never read by the view (`pages/schools/index.html:70-71`); My Plan "Export" points at `/fund-requests?export=csv` — wrong page and a route that redirects (`pages/my_plan/index.html:18-24`); Fund Requests Weekly "Export" same (`weekly.html:493-500`); Planning "Export" — view never reads `export` (`planning/index.html:35-41`); Accounts "Export Bank Payout CSV"/"Export Mobile Money Batch"/"Export Bank File" — no handlers (`accounts/batch_payments.html:53,79,105`). Only **Clusters** export actually works (`cluster_views.py:167-175`) — it is the pattern to copy.

**Dead sidebar navigation per role (bounce → "Access Denied" toast → dashboard):** IA 11 links (Schools, Clusters, Districts, Coverage, Debriefs, Evidence, Reports, Completed Activities, Closure ×2, Analytics Publishing), Partner 10, Accountant 9, RVP 4, HR 3, CCEO 2 (Debriefs, Staff), Coordinator 1. Sidebar gates (`templates/components/sidebar.html`) must be reconciled with `RolePermissionService.can_view_page`. Note the template uses substring matching (`{% if request.user.active_role in 'CCEO,Program Lead,…' %}`) which is fragile (e.g. `"Accountant" in "ProgramAccountant"` is true).

## 2. Dead buttons (no handler, nothing happens)

**108 dead controls across 20 pages** (static analysis, each verified against enclosing forms/Alpine scopes/HTMX). Highest-impact:

- **Finance:** Accounts dashboard — "⚡ Create Disbursement", "📥 Export", detail-pane "Return"/"Hold"/"📥 Download Voucher", pagination (`accounts/dashboard.html:65-381`); **"Submit Monthly Request"** — the page's sole CTA (`accounts/monthly_request.html:41`); Fund Requests Weekly — "✓ Approve All Valid", "Filters", detail-pane "Approve"/"Return" (the real approve exists only inside the request card partial) (`fund_requests/weekly.html:38-295`).
- **Evidence can never be downloaded:** "Download File" is `href="#"` (`accounts/activity_evidence.html:53`); per-evidence "Download" in the packet too (`my_plan/evidence_packet.html:90`).
- **Special Projects dashboard:** entire toolbar dead — "+ New Project", "Import Schools", "Assign Partner", "Add Schools to Project", "Track Impact", "Export Summary" (`dashboards/special_projects.html:100-115`).
- **HR:** "New Staff Member" (`staff/index.html:16`).
- **Completed Activities:** whole filter bar is decorative — selects have no `name`, no form, "Apply Filters" has no handler (`closure/completed_activities.html:20-68`).
- **Reports:** date-range, "Filters", "📥 Export Report" all dead (`reports/index.html:26-38`); Targets same (`targets/index.html:48-52`).
- **CCEO dashboard:** list/calendar/map toggles, three "•••" kebabs with no menu, "+ Add Personal Task" (`dashboards/cceo.html:63-176`).
- ~25 `href="#"` "View all →"-type links across accounts/fund-requests/core-schools/dashboards.
- `static/js/alpine-components.js` is 100% dead code (its `toastManager`/`dropdown`/`drawer` components are never referenced).

**What's healthy:** the drawer system has **zero** orphaned triggers (all 89 `hx-target="#drawer-container"` buttons hit registered endpoints), and all ~110 distinct `hx-get`/`hx-post` URLs resolve against `urls.py`.

## 3. Invisible buttons

- **White-on-white primary CTAs (stale CSS build, root cause #1)** — measured contrast 1.0–1.14:1 in the live browser: **"Add School"** (`/schools`), **"Search"** (`/partners`), **"Create Cluster"** (`/clusters`), **"Verify"** ×45 (IA dashboard), **"⚡ Create Disbursement"** (`/accounts`), **"Take Action"** (fund requests), **"Customize"** (analytics), "+ New Project", "Review Staff Debriefs" (HR), queue tabs "All Pending", pagination numbers, and **the avatar/user-menu button in the header for every role** (`bg-slate-800` missing from bundle → users cannot find profile/switch-role/sign-out).
- **Hover-only row actions are the only way to open records on 9+ pages** (`opacity-0 group-hover:opacity-100`): SSA, Completed Activities, Trainings, Partners, Today, Districts (index + detail), Visits, My Team — measured 99 invisible "View School →" + 19 "Profile →" + 5 "View →" instances live. On touch devices they never appear (still clickable while invisible); only My Team adds `focus:opacity-100` for keyboard users.
- **Touch-hostile menu:** core-schools matrix actions menu opens on `@mouseenter` only — unusable on tablets (`partials/core_schools/matrix_table.html:71`), and uses nonexistent class `hover:bg-slate-55`.
- **Search icon overlaps typed text in the topbar on every page:** the inline rule `input[type="text"] { padding: 8px 12px }` (`base.html:132`) beats the `pl-9` utility on specificity — computed padding-left is 12px while the icon ends at 28px.
- Low-contrast at small sizes: status badges (500-tone text on 50-tone bg ≈ 2.2–2.5:1 at 10px uppercase, `components.css:172-196`), drawer close ✕ ≈ 2.4:1 (`drawers.css:111-122`).
- Icon-only buttons missing `aria-label`: table kebabs, topbar bell.

## 4. Content overlap in cards & tiles

- **Signal tiles on the main dashboard:** headings measure scrollWidth 128–149px in 58–70px boxes (`.signal` has `min-height:82px; overflow:hidden` + `whitespace-nowrap`, `pages.css:225-234`) — text clips mid-word and the sparkline SVG collides with the value text (visible on "Needs Attention / 700 Schools").
- **IA dashboard KPI strip bleeds off-screen at 1440px** — "SALESFORCE QUEUE" / "SSA SUBMISS…" tiles clipped with no scroll affordance; the work-queue table clips its last columns ("STA…", status badges cut).
- **Blanket card rule hits non-cards** (`base.html:53-70`): every `.bg-white.rounded-xl/2xl` div gets `hover { transform: translateY(-2px) !important }` + `backdrop-filter` — including table wrappers, the FullCalendar container, and the user dropdown. Click targets shift under the cursor, and backdrop-filter creates stacking contexts that **trap dropdowns/tooltips/popovers inside cards**.
- **Dropdown menus clipped by scroll containers:** my-plan activity table 3-dot menu inside `overflow-x-auto` (`partials/my_plan/activity_table.html:2,146-155`); core-schools matrix dropdown inside `overflow-hidden` (`matrix_table.html:11,85-96`).
- **KPI values clip with no ellipsis:** `components/kpi_card.html:27-29` + `.card { overflow:hidden }` — full UGX amounts (e.g. CD dashboard `cd.html:32`) silently truncate at ≤1280px in 4-column grids.
- **Sidebar active-indicator bug:** `.app-sidebar__item--active::before` is absolutely positioned but `.app-sidebar__item` never sets `position:relative` (`components/sidebar.css:88-101,167-176`) — on desktop the indicator resolves against the viewport.
- **Z-index traps:** the drawer wrapper's `relative z-50` (`components/drawers/base_drawer.html:28`) flattens its internal z-100/101 to 50 vs page siblings — toasts vs drawers vs the planning bulk-bar ordering currently works only by DOM-order luck; FullCalendar's z-9999 popover is trapped in the calendar card's stacking context and slides under the sticky topbar.
- Filter-chip carets overlap their labels at tablet width (observed at 820px: "Quarter: Q4", "Activity Type: All").
- At exactly 768px, Tailwind `md:` shows the 260px sidebar AND jumps grids to 4 columns (~103px KPI cards); `drawers.css:149` (`max-width:768px`) overlaps Tailwind's `min-width:768px` — desktop layout + mobile bottom-sheet drawers simultaneously.

## 5. Dead space

- **33 of 70 audited pages end in 240–642px of empty scroll** (worst: debriefs, quality-checks, partner pages, IA sub-pages at 640+px) — mostly short/empty content with no empty-state component, plus `min-h-screen` used inside the already-viewport-height `<main>` (adds a guaranteed ≥56px void: `core_schools/index.html:7`, `champions.html:7`, `champion_candidates.html:7`, `admin/user_detail.html:7`).
- **Inconsistent page gutters:** `dashboards/main.html:7` has no padding at all (content touches sidebar/topbar) while sibling pages use `p-6` with max-widths that jump between 7xl / 1400 / 1500 / 1600 / 1680px page-to-page. 14 pages use `max-w-[1600px]`, which is one of the classes **missing from the CSS bundle**, so those pages render unconstrained full-bleed.
- Auto-fill grids reserve empty tracks when items are few (`.summary-strip`, `.mini-metrics`, `pages.css:91-102`); `dashboards/main.html` renders its panel scaffolding unconditionally — bordered empty panels when data is absent.
- Two parallel empty-state systems (`.edify-empty-state` vs `components/empty_state.html`) → inconsistent look.
- **Orphaned CSS:** `edify-tokens.css`, `edify-components.css`, `edify-pages.css` are never loaded (near-duplicates with different values — editing them silently does nothing), and ~2,500 lines of dead hand-written CSS ship inside `main.css` (`mfr-*`, `workflow-*`, `.dark`/`.glass` themes — zero template usage).

## 6. Filter functionality

**Verdicts per surface (full detail in the tables below): Analytics WORKS; Fund Allocation WORKS; Partners/Districts/SSA/Visits/Trainings WORK; Clusters PARTIAL; Schools PARTIAL; Core Schools PARTIAL; Staff PARTIAL; IA Queue PARTIAL (3 of 6 dead); Planning PARTIAL; My Plan DEAD.**

Three systemic defects:

1. **[NO-SUBMIT] My Plan's filter bar sends no request at all.** The form has `hx-get` but no `hx-trigger`, no submit button; the selects have `hx-trigger="change"` but no verb (not inherited by children in htmx). Backend supports every param — this is a one-attribute fix (`partials/my_plan/filters.html:2-8`; view `my_plan/services.py:290-368`).
2. **[DUP-PARAM] Tab switching is broken on Schools and My Plan.** Tabs use `hx-get="…?tab=X"` + `hx-include="#filters-form"` while the form holds a hidden input with the same name; htmx appends included params after the URL's own, Django takes the **last** value, so the stale hidden value wins and clicking a tab re-runs the current tab (`partials/schools/tabs.html:5-49` + `pages/schools/index.html:89`; `partials/my_plan/period_tabs.html:7-44` + `filters.html:10`). The correct pattern (set hidden input, dispatch change) already exists in `matrix_table.html:338-367` and `planning/index.html:144-172` — copy it.
3. **[ID-SPACE] Staff filters silently return zero rows.** Selects submit `User.id` while the filtered columns store `StaffProfile.id`: Core Schools (`partials/core_schools/filters.html:52` vs `core_planning_services.py:75-77`), Planning (`planning/index.html:104` vs `planning_service.py:149-150`), My Plan (same class of bug). To a tester this looks like "filter wipes my data".

Other broken/dead filter controls:
- **IA Verification Queue:** FY select sends `FY25` but `Activity.fy` stores `"2025"` → always zero rows (and `FY25` renders pre-selected while the list is unfiltered — the UI lies); District and Staff are free-text name inputs compared against CUID id columns → always empty (`pages/ia/verification_queue.html:161-212` vs `ia_views.py:65-78`).
- **Dead selects:** Schools `fy` (read, never applied — `school_views.py:92`), Clusters `quarter` (read, never used — `clusters/services.py:691`), Planning `quarter` (same), Staff "Any Status" (no name/handler), all dashboard dropdowns (CD FY/"Uganda", CCEO "Sort by: Priority", Special Projects search/month/region — all nameless with no handlers).
- **Planning "Clear Filters" throws a TypeError** (references a `q` input that doesn't exist on the page) and does nothing (`planning/index.html:123`).
- **Fund Requests Weekly `q` search box:** view never reads `q` (`weekly.html:561`; no `GET.get("q")` in `budget_views.py`).
- Analytics "Clear Filters" hardcodes `?fy=2026&quarter=Q2` (`partials/analytics/filters.html:111`); Fund Allocation export drawer drops active region/search filters from the CSV.
- Staff tabs drop the active search query (plain `href="?tab=X"`).

**Healthy patterns worth keeping:** pagination preserves filters on Clusters, Schools, Core Schools, Planning, and Fund Allocation; Clusters export serializes the live filter form; Analytics' form-level `change` trigger + OOB KPI swaps work well.

## 7. Other findings

- **Demo-data/FY misalignment:** with the seeded demo DB, a CCEO's My Plan and dashboard KPIs show all zeros (default filter lands on FY2026/Q4/July while seeded activities sit elsewhere). Pilot users must not land on empty screens — align seed FY with `get_operational_fy()` or default filters to a period with data.
- Partners list shows "—" for every region (seed data lacks `region_name`).
- The `/leave-requests`, `/staff` crashes plus HR having only ~11 nav items mean **HR currently has almost no working surface.**
- `body` classes reference undefined vars (`--bg-page`/`--foreground` are defined only in orphaned/duplicate files — value comes from the inline body rule by luck).
- Fonts: `--font-sans` in the compiled bundle references `--font-source-sans` which is never defined (Outfit loads separately via Google Fonts; on CDN failure everything falls to system fonts).

## 8. Prioritized fix plan

**P0 — before any user touches it (≈3–5 dev-days):**
1. **Rebuild the CSS pipeline**: add Tailwind CLI (standalone binary works without Node in CI), a `tailwind.config` scanning `templates/**/*.html`, regenerate `main.css`, and add a CI check. This alone fixes every white-on-white button, the avatar menu, page max-widths, and collapsed chart containers.
2. Fix the 4 crashing views (`dashboard_views.py:152` — use `StaffSchoolAssignment` two-step lookup; `staff_views.py:90`; `extended_views.py:596`; coverage view) + the IA `month` latent crash.
3. Fix Accountant dashboard `x-data` JSON injection (`json_script` pattern).
4. **Remove or clearly gate all hardcoded mock data** in `finance_operating_views.py`, `ia_views.py`, `budget_views.py:496`, `extended_views.py:734` — wire real queries or hide the page from the pilot. A finance tool showing fake fund requests to a real accountant is a trust-killer.
5. Replace the 6 alert()-only buttons with real actions or remove them (worst: admin "Save System Changes").
6. My Plan filter form: add `hx-trigger="change"` (one attribute). Fix Schools/My Plan tab [DUP-PARAM] using the existing hidden-input+dispatch pattern.
7. Reconcile sidebar role gates with `can_view_page` (delete ~40 dead nav links across roles).
8. Wire the topbar search to GET `/search` on non-schools pages (the results page already works).
9. **Vendor HTMX/Alpine/ApexCharts/Flowbite into `static/`** (they're npm packages; pin the exact versions) so field users on flaky networks don't lose all interactivity.
10. Fix search-input padding (scope the inline `input[…]{padding}` rule or add `!important` to the utility) and fix `.app-sidebar__item` `position:relative`.

**P1 — during the pilot window (≈1 week):**
- ID-space staff filters (decide one id space, fix writers or selects) and the IA queue's three dead filters (FY value format; district/staff selects instead of free-text-vs-CUID).
- Kill or wire the 108 dead buttons on pilot-scoped pages (evidence Download links first — they block a real workflow; then finance CTAs; then exports, copying the working Clusters pattern).
- Fix Champion workflow routing (`urls.py` order + `frontend:` namespacing) or remove it from scope.
- Give hover-only row actions a visible at-rest state (or always-visible on `(hover:none)`), add `focus-visible:opacity-100`; make the matrix menu click-toggled.
- Empty states for every list page (fixes most of the 33 dead-space voids); standardize on one empty-state component.
- Scope the blanket `.bg-white.rounded-xl` card rule to a real `.card` class; remove `overflow:hidden` from `.card` or add truncation+`title` to KPI values.

**P2 — post-pilot cleanup:**
- Delete orphaned CSS files (`edify-tokens/components/pages.css`), dead `main.css` tail, dead `alpine-components.js`; consolidate `.btn`/`.card`/input definitions into one source; establish a z-index scale; fix the 768px breakpoint collision (`drawers.css:149`) and the 768–1024 grid regime; accessibility pass (aria-labels, badge contrast); replace substring role checks in the sidebar with list membership.

## 9. Pilot rollout proposal

1. **Scope the pilot to the CCEO and Program Lead journeys** (My Plan → schedule → start → evidence → submit → PL review), plus Schools/Clusters directories. These surfaces are the most complete: drawers, HTMX actions, and pagination all verified working. Optionally add Partner field officers (their pages render, but 10 dead sidebar links need removing first).
2. **Hold back from the pilot:** Accountant dashboard, IA Quality Dashboard, CD dashboard, Special Projects, Reports/Targets, HR pages, Champion workflow — every one currently contains mock data, crashes, or an entirely dead action surface.
3. **Before invite:** run the P0 list, then re-run this audit's crawl (scripts are reproducible) as a regression gate: zero 500s, zero white-on-white controls, zero alert()-only actions on scoped pages.
4. **Seed a staging DB with realistic data in the operational FY** so first-session screens aren't empty; verify each pilot role's landing page shows non-zero, truthful numbers.
5. **Instrument the pilot:** keep the existing audit log of `unauthorized_page_access` (it will confirm the sidebar cleanup worked), add basic page-view logging, and give testers a one-tap feedback channel.
6. **Test on the devices users actually have** — at least one Android phone and one tablet; the hover-only affordances and 768px issues will not show on developer laptops.
